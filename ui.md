## 1. Термины

* **Проект** - папка gamedata, необходима чтобы можно было индексировать все файлы и секции
* **Файл** - файл формата `ltx` в котором лежат секции

## ?. Возможности редактора

1) Глобальная история действий в рамках проекта с возможность undo/redo изменений
    2) Должна сохраняться если проект был закрыт и восстанавливаться если проект открыт
3) Должна быть возможность скопировать значение этого поля и потом вставить его в другое этого же типа поле обновив тем
   самым значение
4) Горячие клавиши с возможностью перемещения по файлам как в JetBrains

## ?. Рабочий процесс

### ?.1. Рабочий процесс с проектом

1) Была выбрана папка `gamedata`
2) Произошло индексация проекта
3) Произошла инициализация рабочего пространства
    4) Доступные файлы в дереве файлов проекта

### ?.2. Рабочий процесс с файлом

1) Открыт только файл
    2) В этом случае должен эмулироваться проект так как будто в нем только этот файл
3) Следовать [процессу](#1-рабочий-процесс-с-проектом) с проектом

### ?.3. Типы полей

* **Прогрессия**
    * когда нажимаешь +, то добавляется 2 поля для ввода: следующая граница и следующее значение
* **Переводимая строка**
    * Должно быть 2 варианта заполнения:
        1) Ввод просто текста, что предположительно является строкой
        2) Выбор из списка строк, для которых есть перевод
* **Иконка**
    * Должно быть 2 варианта заполнения:
        1) Ввод просто текста, что предположительно является именем иконки
        2) Выбор из списка иконок

### ? Объявление типов полей

1) Указать тип поля
2) Если возможно, то определиться со значением `isList`

### ? Создание новый типов полей

1) У каждого типа есть флаг `canBeList`, который определяет может ли поле такого типа быть списком
