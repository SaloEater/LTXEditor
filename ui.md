Вопросы

* **Может ли быть несколько секций с одинаковым именем?**

## 1. Термины

* **Проект** - несколько одновременно используемых папок gamedata, необходим чтобы можно было индексировать все файлы и
  секции
* **Файл** - файл формата `ltx` в котором лежат секции

## Возможности редактора

* Переключение между текстовым и графическим представлением секций?

1) Проект
    1) Создать новый проект
    2) Открыть существующий проект
    3) Закрыть проект
2) gamedata
    1) Добавить папку gamedata в проект
    2) Убрать папку gamedata из проекта (ПКМ по папке в дереве файлов)
3) Глобальная история действий в рамках проекта с возможностью undo/redo изменений
    1) Перемещение секции по файлу
    2) Переименование секции
    3) Изменение типа секции
    4) Изменение типа поля
    5) Переименование поля
    6) Изменение значения поля
    7) Изменение настроек в окне свойств справа
    8) Перемещение полей по секции
    9) Создание новой секции
    10) Удаление секции
    11) Создание нового поля
    12) Удаление поля
    13) Создание файла
    14) Удаление файла
    15) Добавление родительской секции
    16) Удаление родительской секции
    99) Было бы круто если история будет сохраняться когда проект был закрыт и восстанавливаться когда проект открыт
4) Копирование
    1) Можно скопировать секцию
        1) Можно вставить в другую секцию тем самым перезаписывав ее
        2) Можно сделать дубликат
    2) Можно скопировать поле
        1) Но копируется только значение, которое можно вставить в другое поле такого же типа
    3) **!!!** Подумать, нужна ли массовая замена
5) Переименование секций
    1) Переименование секции должно быть глобальным, то есть если секция была переименована, то все ссылки на нее
       должны обновиться на новое значение
6) Переименование полей
    1) Выдается контекстное меню с выбором
        1) Переименовать поле во тип секции и во всех секция этого типа
        2) На основе старого типа секции сделать новый тип с переименованным полем
7) Горячие клавиши
    1) Перемещение по файлам
        1) История перемещения
            1) Здесь отслеживается история с какими полями > секциями > файлами взаимодействовал пользователь. Это
               хорошо
               сделано в JetBrains IDE, где с помощью CTRL+ALT+ARROW можно перемещаться по местам где был написан код
        2) Перемещение по секциям (Глобально)
            1) CTRL+клик по секции в поле проваливается в эту секцию
            2) CTRL+B после нажатия на заголовок секции открывает меню со списком секций которые на нее ссылаются
        3) Перемещение по секциям (В рамках файла)
            1) Возможность переключаться на
                * следующую секцию
                * предыдущую секцию
                * первую секцию
                * последнюю секцию
        4) Переключаться между вкладками
            1) Вкладка слева
            2) Вкладка справа 
    2) Создание/сохранение/открытие файлов
    3) Удаление поля
    4) Удаление секции
    5) Создание дубликата секции
    6) Создание новой секции этого же типа 
8) Наследование секций
    1) Если одна секция наследует другую, то она автоматически получает список полей родительской секции
    2) Если тип у секции-наследника отличается от родителя, то считается что у этой секции есть эти поля и поля ее
       родителя
    3) Поля секции-наследника "перезаписывают" поля родителя в интерфейсе
9) Файл
    1) Можно создать новый файл
    2) Можно открыть существующий файл
    3) Можно закрыть файл
        4) Если файл был изменен, то при закрытии будет предложено сохранить изменения. По умолчанию не сохранять
           изменения
    4) Можно найти все скрипты которые используют этот файл 
10) DLTX
    1) Поддержка формата DLTX
        1) Когда секция назначается как `overwritten by dltx`, то для ней должен быть автоматически вычислен путь до 
root file

## Рабочий процесс

### Рабочий процесс с проектом

1) В проект были добавлены папки `gamedata`
2) Произошло индексация проекта
3) Произошла инициализация рабочего пространства
    1) Доступные файлы в дереве файлов проекта
    2) Если есть сохранение, то открыты те файлы, которые были открыты при прошлом закрытии проекта

### Рабочий процесс с файлом

1) Если открыт файл из активного проекта, то все происходит как будто файл был выбран из дерева
2) Если открыт файл не из активного проекта, то он просто добавляется в рабочее пространство, индексируется,
   но не сохраняется в проект и не отображается в дереве файлов

### Изменения типа секции

1) Контекстное меню с выбором
    1) Добавить недостающие поля - Да/Нет
    2) Если Да, то добавляют все недостающие поля в секцию
2) Все поля которые не совпадают по типу и имени - удаляются
3) У любой секции может быть выбран тип "Неизвестно"
    1) Это специальный тип при котором
        1) Нет никаких валидации
        2) Все поля имеют тип "Неизвестно"
3) Должна быть возможность отслеживать и удалять неиспользуемые типы секций

### Изменения типа поля

1) Перестраивается интерфейс значения поля на новый тип, старое значение удаляется

### Типы полей

* **Прогрессия**
    * когда нажимаешь +, то добавляется 2 поля для ввода: следующая граница и следующее значение
* **Переводимая строка**
    * Должно быть 2 варианта заполнения:
        1) Ввод просто текста, что предположительно является строкой
        2) Выбор из списка строк, для которых есть перевод
* **Иконка**
    * Должно быть 2 варианта заполнения:
        1) Ввод просто текста, что предположительно является именем иконки
        2) Выбор из списка иконок

### Закрытие проекта

1) Сохранение списка всех открытых файлов
2) Сохранение истории действий
